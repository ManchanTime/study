# 의존관계 주입 방법
1. 생성자 주입
  + 생성자 호출시점에 딱 1번만 호출되는 것이 보장된다.
  + 불변, 필수 의존관계에 사용
   
2. 수정자 주입(setter 주입)
  + 선택, 변경 가능성이 있는 의존관계에 사용
  + 자바빈 프로퍼티 규약의 수정자 메서드 방식을 사용하는 방법이다.
  
3. 필드 주입
  + 코드가 간결해서 많은 개발자들을 유혹하지만 외부에서 변경이 불가능해서 테스트 하기 힘들다는 치명
적인 단점이 있다.
  + DI 프레임워크가 없으면 아무것도 할 수 없다.
  + 사용하지 말자.
  
4. 일반 메서드 주입
   + 한번에 여러 필드를 주입 받을 수 있다.
